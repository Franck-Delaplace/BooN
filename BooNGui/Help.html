The graphical interface allows the creation and the analysis of Boolean networks called BooN through two classes of methods: file and network. The file methods manage files containing networks, while the network methods analyze the network. The network methods are also available in the tool bar at left hand side of the window. The last Help menu includes: [Undo], [Redo] and [Help] functionalities.

Design network

The network is designed interactively. At the bottom of the graph window, there is a pattern network which is used to select the edge style, either positive (Green) or negative (Red).  This network will never be included in the final BooN. Click on the appropriate edge in this network and all new edges will be of the same type as this edge. The selected node or edge is always emphasized.
The network design commands are:

  • Create a node: press [insert] or + will add a new node to the graph at mouse position.
  • Create an edge: To create and style a new edge, first select an edge with the desired sign or color, then double-click the starting node. Double-click the destination node next.
  • Delete a node or edge: select the node or edge and press [delete] or -.
  • Change edge direction: pressing [@] will reverse the direction of the selected edge.
  • Label node or edge : select the node or edge, press [enter] , type the label, and press [enter] again. A label must follow the syntax of a variable.

INFO

  • The nework desing is based on EditableGraph function of Netgraph module. Please refer to the documentation of this function for the commands related to the graph design.
  • To prevent the GUI from becoming unresponsive, remember to press [enter] a second time to complete the labeling process. If forgotten, select the node or the edge where the label was modified again and press [enter] to end the labeling.
  • Nodes will only be integrated if they are named, so label a node is necessary to include it in the BooN.
  • Edge labels correspond to module ID and must be integers, otherwise they are not considered. If a variable occurs in several modules, separate the integer label of each module by a space. If no labels is set for an edge, then the source node is implicitly included in the first module (ID = 1).
  • The default formula for a newly created node is False when it is isolated.
  • It is also possible to create a network by directly specifying the formula related to a node (See View).

File Menu

You can load, save a BooN and import or export it. The exportation or importation is defined from a text file specification of a BooN. The extension of a BooN file is .boon and it is accessible by [Load] and [Save]. A text file specifying a Boolean network can be imported [Import] or exported [Export] with .txt as default extension.

INFO:
The coordinates of the nodes are lost during the exportation, to keep them save the BooN before.

Syntax of the imported text BooN file

The file contains a set of equation where a Boolean equations must be written in the same line.  Comments can be added and must start by  #.  The formula must follow the Sympy syntax, namely: (AND = &, OR = |, NOT = ~).

The grammar of an equation is : <variable> '=' <formula>. For example, we have:

EXAMPLE:
# Example of BooN
x = (~x & y) | z
y = ~z | x
z = y & x & ~z

Boolean Network Analysis

The different items are:
  • [View BooN] shows the Boolean network specification corresponding to the current BooN. conversion to DNF is available via this functionality. Moreover, the formulas can be redefined by typing a new formula in the formula field of a variable.
  • [Stable states] compute the stable states of the BooN. The method is based on SAT Solver and can be applied to large network.
  • [Controllability] perform a controllabillity analysis to determine which variables must be frozen to reach the expected goal identified by a variable marking.
  • [Model and Equilibria] Compute the dynamics following the asynchronous or synchronous mode. In asynchronous mode, only one variable is updated per transition, whereas in synchronous mode, all variables are updated jointly. We suggest you switch to Python style first to make these changes

INFO:
The model of dynamics is exponential and thus this computation is reduced to a given number of variables.
The syntax of a new formula typed in the formula field must follow the SYMPY/Python syntax whatever the current output style, and only use the already drawn nodes. Switching to Python style first is advisable to make these changes.

Controllability analysis

Controllability analysis determines the variables that must be fixed to achieve a specific goal. The goal is either to reach or avoid a Boolean profile of variables as markers at steady-state. The markers are purposely selected variables. The procedure is decomposed into the following steps:

Define marker profile:
On the left-hand side table, select the destination profile by selecting value of markers. On the left-hand side table, select the destination profile by setting the markers to true or false.
Observers characterization:
The observers are variables that cannot be controlled by definition. Set the observers in the second tab. The markers are always observers.
Choose the nature of the objective:
Choose either to [Reach] or [Avoid] the marking.
Determine the modality of the objective
There are two modalities: [Possibility]and [Necessity]. In the case of a marking profile to reach, possibility implies the existence of this marking profile in a stable state at least. Necessity dictates that all stable states possess this profile. The same applies to avoidance. However, for avoidance, necessity alone can generate solutions without stable states. To achieve a stable state solution, it is necessary to consider both possibility and necessity.
Find the possible control:
Click on the control button to compute the different possible solutions which are displayed at right.
Apply the control:
Select one control solution and click on the rightmost button to apply it to the BooN. The BooN will be modified, and the controllability analysis window will close.
